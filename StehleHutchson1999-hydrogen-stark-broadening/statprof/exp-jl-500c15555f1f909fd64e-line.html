178 # Method for Float64
179 # 1. Argument reduction: Reduce x to an r so that |r| <= log(b, 2)/512. Given x, base b,
180 # find r and integers k, j such that
181 # x = (k + j/256)*log(b, 2) + r, 0 <= j < 256, |r| <= log(b,2)/512.
182 #
183 # 2. Approximate b^r-1 by 3rd-degree minimax polynomial p_b(r) on the interval [-log(b,2)/512, log(b,2)/512].
184 # Since the bounds on r are very tight, this is sufficient to be accurate to floating point epsilon.
185 #
186 # 3. Scale back: b^x = 2^k * 2^(j/256) * (1 + p_b(r))
187 # Since the range of possible j is small, 2^(j/256) is stored for all possible values in slightly extended precision.
188

194 # 2. Approximate b^r by 7th-degree minimax polynomial p_b(r) on the interval [-log(b,2)/2, log(b,2)/2].
